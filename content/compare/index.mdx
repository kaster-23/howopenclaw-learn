OpenClaw is a self-hosted AI agent you run on your own hardware and access through the messaging apps you already use — Telegram, WhatsApp, iMessage, Slack. That positioning puts it in an interesting spot: it overlaps with automation tools, developer APIs, chatbot platforms, and general AI assistants, but isn't exactly any of them.

These pages break down the key differences honestly — including situations where another tool is the better fit.

## Comparisons

| Tool | Why people compare it | Who wins |
|---|---|---|
| [Claude Code](/compare/openclaw-vs-claude-code) | Similar brand names ("Claude" / "Claw") cause confusion | Different tools — Claude Code is for coding, OpenClaw is for daily personal use |
| [ChatGPT](/compare/openclaw-vs-chatgpt) | Both are AI assistants you can chat with | ChatGPT for casual use; OpenClaw when you want privacy, your own data, and messaging integration |

## What OpenClaw Is

OpenClaw runs a Gateway on your machine — a background process that connects to your messaging channels, loads your personal context files (SOUL.md, USER.md, HEARTBEAT.md), and routes conversations to your chosen AI provider (Claude, GPT-4, Gemini). You configure it through JSON and plain-text markdown files. No drag-and-drop editor, no hosted service, no subscription.

The result is an AI that behaves like yours: it knows your preferences, maintains memory across conversations, runs scheduled automations (daily briefings, reminders), and responds through the messaging app you're already in.

## When OpenClaw Is Not the Right Tool

- You want a coding assistant → use [Claude Code](/compare/openclaw-vs-claude-code) or GitHub Copilot
- You need to connect 100+ SaaS apps → use [Zapier](/compare/openclaw-vs-zapier) or [n8n](/compare/openclaw-vs-n8n)
- You're building an AI product for other people → use [OpenAI Assistants API](/compare/openclaw-vs-openai-assistants)
- You want zero setup and don't care about privacy → use [ChatGPT](/compare/openclaw-vs-chatgpt)

If you've decided OpenClaw is the right fit, the [installation guide](/foundation/install) covers setup from scratch.
