OpenClaw agents have significant system access — shell, files, web, messaging. That's what makes them powerful. It's also why security guardrails matter.

> **The line between an autonomous agent and a dangerous one isn't a prompt. It's a design choice.**

## Why Security Matters More Than You Think

OpenClaw is a persistent, autonomous agent — not a chatbot. It can:

- Access and modify files
- Execute shell commands
- Browse the internet
- Send messages on your behalf
- Install and run skills
- Use your API keys

Misconfiguration creates real attack surfaces: prompt injection, unauthorized file access, credential theft, message spoofing, and malicious skill installation. OpenClaw has built-in security primitives — but you have to configure them correctly.

## DM Policy: Who Can Talk to Your Agent

Three policy modes control who can initiate contact with your agent:

### Allowlist (Recommended)

Only explicitly approved contacts can message the agent. Unknown senders are silently ignored — no error, no acknowledgment, nothing.

**Best for:** iMessage, WhatsApp, Signal, and other personal channels.

### Approval (Middle Ground)

Unknown contacts trigger an approval request sent to you. You approve or deny before the agent responds.

**Best for:** Telegram, Discord, or any channel where you want to stay in control but allow new contacts.

### Open (Use With Extreme Caution)

Anyone can message. No restrictions whatsoever.

<Callout type="error" title="Open policy is almost never safe for a personal agent">
With Open policy, anyone who finds your agent's contact can send it commands. Combined with shell access, file tools, or messaging capabilities, this is a serious and exploitable vulnerability. Never use Open on a personal channel.
</Callout>

**Nearly never appropriate** for personal channels. Only consider this for a dedicated, heavily sandboxed public bot with explicit restrictions in SOUL.md.

## Policy Recommendations by Channel

| Channel | Recommended Policy |
|---|---|
| iMessage | Allowlist |
| WhatsApp | Allowlist |
| Signal | Allowlist |
| Slack (workspace) | Approval or Open |
| Telegram | Approval |
| Discord | Approval |
| WebChat | Open |

## Pairing: The Trust Handshake

Pairing establishes a trusted relationship between your agent and a specific contact. It's how your agent knows which humans to trust.

### How Pairing Works

1. Contact messages your agent (or you initiate pairing manually)
2. If your policy requires it, you approve the contact
3. Your agent generates a short pairing token
4. The contact confirms the token
5. The contact is now paired

### What Paired Contacts Can Do

- Send direct messages for your agent to process
- Be referenced in message routing rules
- Maintain a separate conversation history
- Receive proactive messages from your agent

### What Paired Contacts Cannot Do

- Access your files or system
- See other contacts' conversations
- Modify your agent's settings or skills
- Bypass tool restrictions

## When to Pair

**Pair:** Family, close friends, trusted colleagues, and your own devices. Anyone you'd trust with an API key.

**Don't pair:** Unknown people, contacts from untrusted channels, anyone with elevated agent permissions you haven't manually verified.

## Configuration

Set your DM policy in `~/.openclaw/openclaw.json`:

```json
{
  "channels": {
    "imessage": {
      "dmPolicy": "allowlist"
    },
    "telegram": {
      "dmPolicy": "pairing"
    }
  }
}
```
