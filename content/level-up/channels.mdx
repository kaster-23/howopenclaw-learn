## Telegram

Already covered in [Connect Your First Channel](/foundation/first-channel). The simplest setup.

Multiple Telegram users are supported by adding their IDs to `allowlist`. You can also have multiple bots (one per project) by configuring multiple channel entries.

## WhatsApp

WhatsApp uses a QR code pairing approach — no official API key required.

```json
{
  "channels": {
    "whatsapp": {
      "dmPolicy": "pairing",
      "sessionPath": "~/.openclaw/sessions/whatsapp"
    }
  }
}
```

```bash
openclaw channels login --channel whatsapp
```

Scan the QR code with your phone. The session is persisted so you only need to pair once.

**Important:** WhatsApp uses an unofficial bridge (Baileys). This is stable but WhatsApp could theoretically ban the number. Use a secondary number if this is a concern.

## BlueBubbles (recommended for iMessage)

The recommended way to connect iMessage. BlueBubbles uses the BlueBubbles macOS server REST API to bridge iMessage to OpenClaw.

```json
{
  "channels": {
    "bluebubbles": {
      "dmPolicy": "allowlist",
      "allowlist": ["your.email@icloud.com", "+1234567890"]
    }
  }
}
```

```bash
openclaw channels setup bluebubbles
```

You'll need the BlueBubbles server running on a Mac. Follow the permission prompts. Once configured, any iMessage to the addresses on your allowlist will reach the agent.

Works with iPhone's Messages app and iMessage on Mac. Not compatible with Android SMS.

Covered in [Set Up Slack](/goals/work-hub/slack). Requires a Slack app with Socket Mode.

The Slack integration supports:
- DMs to your bot
- `@mentions` in channels the bot is added to
- Posting to specific channels via automations

## Discord

Similar to Slack — requires creating a Discord bot.

1. Go to [discord.com/developers/applications](https://discord.com/developers/applications) → New Application
2. Go to Bot → Enable "Message Content Intent"
3. Copy the Bot Token
4. Go to OAuth2 → URL Generator → select `bot` scope + `Send Messages`, `Read Message History`
5. Use the generated URL to invite the bot to your server

Config:

```json
{
  "channels": {
    "discord": {
      "token": "YOUR_DISCORD_BOT_TOKEN",
      "dmPolicy": "allowlist",
      "allowlist": ["YOUR_DISCORD_USER_ID"]
    }
  }
}
```

To find your Discord user ID: Settings → Advanced → Enable Developer Mode → right-click your username → Copy User ID.

## Multiple channels simultaneously

You can run all channels at once:

```json
{
  "channels": {
    "telegram": { ... },
    "slack": { ... },
    "whatsapp": { ... }
  }
}
```

Each channel has its own pairing/allowlist. The agent maintains separate conversation contexts per channel but shares SOUL.md, USER.md, and MEMORY.md.

For routing specific automations to specific channels, use the `channel` field in each automation config.
