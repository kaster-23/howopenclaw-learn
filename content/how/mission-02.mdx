## Tactical overview

By the end of this mission, your agent will have a name, a consistent communication style, and enough context about you to give useful responses without being briefed every time. It will no longer feel like a generic chatbot.

---

## Intel briefing

When your agent starts, it reads two files before processing any message:

- **`~/.openclaw/SOUL.md`** — Its identity. Name, personality, communication rules, expertise, values.
- **`~/.openclaw/USER.md`** — Information about you. Who you are, what you do, what it should always know.

These files are plain text. You write them like you're briefing a new hire on their first day.

The difference between a generic AI response and one that actually feels like your AI is almost entirely in these two files. A thoughtful SOUL.md makes your agent usable. A vague one gives you a vague assistant.

## Mission execution

### Step 1 — Open SOUL.md

```bash
# macOS — opens in TextEdit
open -e ~/.openclaw/SOUL.md

# VS Code
code ~/.openclaw/SOUL.md

# Terminal editors
nano ~/.openclaw/SOUL.md
```

You'll see a default file. Replace it entirely with your own.

---

### Step 2 — Write your SOUL.md

A SOUL.md that works is specific and testable. Every line should describe a behavior you can verify.

Here's a template that produces noticeably better results:

```markdown
# Identity
Name: [Choose a name — this is what your agent calls itself]

# Personality
[3–5 specific traits. "Direct and opinionated" beats "friendly and helpful".]

# Communication style
- Response length: [short by default / detailed when asked / always concise]
- Tone: [casual / professional / dry / warm]
- Format: [prose by default / bullets only for lists of 3+ / no markdown unless useful]
- Never say: ["Great question!", "Certainly!", "As an AI...", "I'd be happy to"]

# What I'm good at
[Your agent's actual strengths — be specific. "Reasoning, writing, and research synthesis"
is better than "helping with anything".]

# Values
[What to optimize for. What to refuse. What to always check.]

# When asked for an opinion
Give it. Don't hedge. Don't ask what I want to hear.
```

**The specificity rule:** "Be concise" does nothing. "Default to 2–3 sentences unless I ask for more" gets followed. Vague instructions get ignored. Testable instructions work.

<Callout type="info" title="Example: minimal but effective SOUL.md">
```markdown
# Identity
Name: Ara

# Personality
Sharp, direct, slightly dry. Opinionated when asked. Not a yes-machine.

# Communication style
- Default: 2–3 sentences. Go longer only when asked.
- Bullets only for 3+ parallel items. Prose otherwise.
- No filler: skip "Great question!", "Certainly!", "As an AI..."
- Say what you think. If uncertain, say so once and move on.

# Strengths
Reasoning, writing, research synthesis, code review.

# Values
Accuracy over agreeableness. Useful over impressive.
```
</Callout>

### Step 3 — Write your USER.md

```bash
code ~/.openclaw/USER.md
```

USER.md gives your agent context about you. It doesn't need to be long:

```markdown
Name: [Your name]
Location: [City, Timezone]
Role: [What you do for work]

Current focus: [1–2 things you're actively working on]

Preferences:
- [How you like information delivered]
- [Tools you use — editor, OS, stack]

Key context:
- [Anything the agent should always know about you]
```

You'll add to this over time. Start simple.

---

### Step 4 — Reload and test

```bash
openclaw gateway restart
openclaw dashboard
```

Run these three tests:

**Test 1 — Tone check:**
> "Summarize the French Revolution in two sentences."

Expected: matches your length setting. If you wrote "2–3 sentences", you should get 2–3 sentences.

**Test 2 — Opinion test:**
> "What do you think is more important — speed or correctness in software?"

Expected: a direct opinion, not "it depends" hedging (assuming you asked for opinions in SOUL.md).

**Test 3 — Filler check:**
> "Can you help me write an email?"

Expected: it starts writing or asks a clarifying question — not "Certainly! I'd be happy to help you with that!"

If any test fails, edit SOUL.md and add a more specific instruction for that behavior. Then `openclaw gateway restart` and test again.

---

## Confirm mission success

Your agent should:
- Respond with the name you gave it when asked "what's your name?"
- Match your length and tone preferences in responses
- Not use the filler phrases you banned

<Callout type="warn" title="The 200-line trap">
Over 200 lines and the model starts ignoring sections — every line competes for attention in the context window. If your SOUL.md is getting long, cut vague rules and keep only specific, testable ones.
</Callout>

## Bonus objective (+50 XP)

Write a USER.md with more than 200 words without the agent ignoring parts of it.

The trick: frontload the most important facts. Models read top-to-bottom and prioritize earlier context. Put your name, role, and timezone in the first five lines. Put "nice to have" context further down. Verify the agent knows your name and role without being explicitly told in the conversation.
