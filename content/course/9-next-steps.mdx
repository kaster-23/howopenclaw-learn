## Troubleshooting

When something goes wrong, start here:

```bash
openclaw doctor
```

This checks your Gateway, Agent, model connection, and installed skills. It will
tell you exactly what is broken and suggest a fix.

### Common issues

**Agent not responding:**
```bash
openclaw gateway status
openclaw gateway restart
```

**Channel disconnected:**
```bash
openclaw channels list
openclaw channels reconnect telegram
```

**Model errors:**
```bash
openclaw config get model
openclaw models test
```

For a full troubleshooting guide, see the [Reference section](/reference/troubleshooting).

## Keeping OpenClaw up to date

**Update OpenClaw core:**

```bash
openclaw update
```

**Update all installed skills:**

```bash
openclaw skills update --all
```

**Check what's outdated without installing:**

```bash
openclaw skills outdated
```

<Callout type="info" title="How often should I update?">
Check for updates every few weeks. OpenClaw releases frequently with bug fixes
and new features. Skills are updated independently by their authors.
</Callout>

## Getting help

- **Built-in diagnostics:** Run `openclaw doctor` in your Terminal — it diagnoses most issues automatically
- **Documentation:** Browse the [Reference](/reference) section for CLI details and the [Troubleshooting guide](/reference/troubleshooting) for common errors
- **GitHub Issues:** Report bugs or request features on the [OpenClaw GitHub](https://github.com/openclaw/openclaw/issues)
- **Discord:** Join the [OpenClaw community Discord](https://discord.gg/openclaw) for real-time help from other users and contributors

## Contributing

OpenClaw is open source. You can contribute in several ways:

- **Report bugs** — File issues on GitHub with steps to reproduce
- **Write skills** — Share useful skills on ClawHub for others to use
- **Improve docs** — This documentation is open source too
- **Share your setup** — Help others learn from what worked for you
