## Tactical overview

By the end of this mission, your agent will be able to fetch any URL, search the web, and read and write files in your workspace. These are the tools that make it genuinely useful — not just a conversational AI but an agent that can do research, save notes, and interact with information outside its training data.

---

## Intel briefing

Up to now, your agent has been running on internal knowledge — the model's training data and whatever you've put in SOUL.md and USER.md. That's limited. It can't check today's news, fetch a page you send it, or save output to a file you can use later.

Tools change that. OpenClaw includes a set of built-in tools that are off by default. Turning them on gives your agent new capabilities:

- **`web_fetch`** — fetch and read any URL
- **`web_search`** — search the web and return results
- **`read`** — read files from your workspace
- **`write`** — write files to your workspace
- **`exec`** — run shell commands (restricted to your allowlist)

Tools are scoped. `read` and `write` only have access to `~/.openclaw/workspace/`. `exec` only runs commands on your allowlist. Your agent can't go beyond what you explicitly allow.

## Mission execution

### Step 1 — Add the tools block

Open `~/.openclaw/openclaw.json` and add a `tools` section:

```json
{
  "provider": "...",
  "channels": { "..." },
  "automations": [ "..." ],
  "tools": {
    "web_fetch": { "enabled": true },
    "web_search": { "enabled": true },
    "read":      { "enabled": true },
    "write":     { "enabled": true }
  }
}
```

---

### Step 2 — Restart

```bash
openclaw gateway restart
```

---

### Step 3 — Test each tool

Open your dashboard or send messages via Telegram. Run all four tests:

**Test 1 — Web fetch:**
> "Summarize this for me: https://en.wikipedia.org/wiki/Cephalopod"

Your agent fetches the URL and gives you a summary. It should take a few seconds.

**Test 2 — Web search:**
> "What's the current weather in [your city]?"

Your agent searches the web and returns a result with current data.

**Test 3 — File write:**
> "Write a short project brief for my current main project and save it as project-brief.md in my workspace"

Your agent creates the file. Find it at `~/.openclaw/workspace/project-brief.md`.

**Test 4 — File read:**
> "I put a file called notes.txt in my workspace. Organize it into sections."

First drop a file at `~/.openclaw/workspace/notes.txt`, then send the message.

## Understanding the security model

Each tool respects boundaries:

| Tool | Scope | Can it be restricted further? |
|---|---|---|
| `web_fetch` | Any URL | Yes — add a `domains` allowlist |
| `web_search` | Web results via configured search provider | No additional scoping |
| `read` | `~/.openclaw/workspace/` only | No broader access without config change |
| `write` | `~/.openclaw/workspace/` only | No broader access without config change |
| `exec` | Only commands on your allowlist | Allowlist is required |

If you want to add `exec`, always use an allowlist:

```json
"exec": {
  "enabled": true,
  "allowlist": ["ls", "cat", "git status", "git log"]
}
```

Never set `allowlist: []` — an empty allowlist removes all restrictions.

If you want to approve each use before it runs, add `"confirm": true` to any tool:

```json
"exec": {
  "enabled": true,
  "allowlist": ["git commit", "git push"],
  "confirm": true
}
```

With `confirm: true`, your agent will pause and ask you to approve before executing.

- A file appears in `~/.openclaw/workspace/` after asking the agent to write one

## Bonus objective (+50 XP)

Add the `exec` tool with a meaningful allowlist for your actual workflow. If you use git, add `git status` and `git log`. If you use Python, add `python3 --version`. Test it by asking your agent to run one of the commands.

```json
"exec": {
  "enabled": true,
  "allowlist": ["git status", "git log --oneline -10", "ls ~/.openclaw/workspace"],
  "confirm": true
}
```

With `confirm: true`, you'll see exactly what the agent wants to run before it does it.
