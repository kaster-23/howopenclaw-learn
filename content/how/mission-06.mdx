## Tactical overview

By the end of this mission, you'll have an agent that can research any topic across multiple sources and deliver a structured report — on demand or on a schedule. You'll also learn how to use HEARTBEAT.md to give your agent standing context that's always in scope.

---

## Intel briefing

In Mission 05 you enabled tools. In this mission you'll use them together — orchestrating web_search, web_fetch, and write to produce something a single tool call can't do: multi-source research with synthesis.

A research agent isn't a single command. It's a chain:
1. Search for the best sources on a topic
2. Fetch and read those sources
3. Synthesize across all of them
4. Write the result to a file (or send it to a channel)

You also have a second asset to unlock: **HEARTBEAT.md**. This is a file your agent reads every session — not just once during setup, but every time it wakes up. It's how you maintain standing context without repeating yourself in every prompt.

## Mission execution

### Step 1 — Write your HEARTBEAT.md

HEARTBEAT.md lives at `~/.openclaw/workspace/HEARTBEAT.md`. Your agent reads it at the start of every session, so anything you write here stays permanently in scope.

```bash
code ~/.openclaw/workspace/HEARTBEAT.md
```

Write the topics and sources you want your agent to track:

```markdown
# Standing research context

## Topics I track
- [Topic 1 — e.g., AI developments in your field]
- [Topic 2 — e.g., a specific technology, market, or news area]
- [Topic 3]

## Sources I trust
- [Source 1 — name + URL]
- [Source 2]

## Research preferences
- Prefer primary sources over aggregators
- Always include publication date when citing
- Flag anything that contradicts what you knew last week
```

Restart the Gateway so it picks this up:

```bash
openclaw gateway restart
```

### Step 2 — Test a deep research prompt

Send this via Telegram or the dashboard:

> "Research [your topic] using web search. Find 3 credible sources, fetch each one, and write me a structured summary covering: what's new, what's significant, and what I should pay attention to. Save the report to research-report.md in my workspace."

Watch what happens. Your agent will:
1. Run a web search on your topic
2. Fetch each source URL
3. Synthesize across all three
4. Write the file

This takes 30–90 seconds. When it finishes, check `~/.openclaw/workspace/research-report.md`.

<Callout type="info" title="The magic of chained tools">
This works because you haven't written any code. You gave a natural language prompt, and your agent used `web_search`, then multiple `web_fetch` calls, then `write` — automatically, in sequence. This is agentic behavior: planning and executing a multi-step task from a single instruction.
</Callout>

### Step 3 — Schedule a weekly intelligence report

Add a weekly automation to your `openclaw.json`:

```json
{
  "id": "weekly-intel",
  "schedule": "0 9 * * 1",
  "timezone": "America/New_York",
  "prompt": "Weekly intelligence report. Research the latest developments in [your topic areas from HEARTBEAT.md]. Fetch 3-5 recent sources per topic. Write a structured report to weekly-intel.md in my workspace, then send me a 3-sentence summary via this channel.",
  "channel": "telegram"
}
```

This runs every Monday at 9am. Your agent researches, writes the full report to a file, and texts you a summary.

Test it immediately:

```bash
openclaw cron run weekly-intel
```

---

### Step 4 — Sharpen your research prompts

The quality of a research output depends almost entirely on the quality of the prompt. Some patterns that consistently produce better results:

**Specify how many sources:**
> "Find 5 sources, not 3" — more sources = less bias from any single one

**Specify recency:**
> "Only sources published in the last 30 days"

**Specify the output format:**
> "Organized as: Key Findings, Notable Quotes, What This Means, Open Questions"

**Ask for contradictions:**
> "Note any claims that conflict with what was true last month"

**Use your HEARTBEAT.md context:**
> "Given what you know about my interests in HEARTBEAT.md, what from this research is most relevant to me?"
