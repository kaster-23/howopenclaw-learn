## Tactical overview

By the end of this mission, your agent will send you a message every morning at a time you choose — without you opening the app, without you asking, without any manual trigger. This is OpenClaw's automation model: your agent acts on a schedule.

---

## Intel briefing

Everything you've done so far is reactive — you message, the agent responds. Automation inverts that. You define a schedule and a prompt; the agent executes it on its own.

This is the moment an AI assistant becomes an AI agent. The distinction is real: assistants wait. Agents act.

A cron job in OpenClaw has three parts:

1. **A schedule** — when to run, expressed as a cron expression
2. **A prompt** — what instruction the agent receives when the schedule fires
3. **A channel** — where the result gets delivered

The Gateway manages the schedule. Your agent doesn't need you to be at your computer. As long as the Gateway is running, the automation runs.

## Mission execution

### Step 1 — Add the automations block

Open `~/.openclaw/openclaw.json`. Add an `automations` key to your existing config — don't replace the whole file, just add this block:

```json
{
  "provider": "...",
  "channels": { "..." },
  "automations": [
    {
      "id": "morning-briefing",
      "schedule": "0 7 * * *",
      "prompt": "Send me a morning briefing. Include: today's date and day of week, a one-line note on what I should focus on today based on what you know about me, and any standing reminders from my context. Keep it under 8 lines. Plain text, no bullet points.",
      "channel": "telegram"
    }
  ]
}
```

The `provider` and `channels` lines represent whatever is already in your file. You're only adding `"automations"`.

---

### Step 2 — Understand the schedule

`"0 7 * * *"` means: minute 0, hour 7, every day of the month, every month, every day of the week. Your agent runs at 7:00am.

**Cron quick reference:**

| Expression | Meaning |
|---|---|
| `0 7 * * *` | Daily at 7:00am |
| `0 7 * * 1-5` | Weekdays only at 7:00am |
| `30 8 * * *` | Daily at 8:30am |
| `0 8,18 * * *` | Daily at 8am and 6pm |

<Callout type="warn" title="Running on a server or VPS?">
Servers run in UTC by default. If your 7am briefing arrives at noon, your system clock is UTC. Fix it by adding a timezone to the automation:

```json
{
  "id": "morning-briefing",
  "schedule": "0 7 * * *",
  "timezone": "America/New_York",
  "prompt": "...",
  "channel": "telegram"
}
```
</Callout>

---

### Step 3 — Reload

```bash
openclaw gateway restart
```

Verify your automation loaded:

```bash
openclaw cron list
```

You should see `morning-briefing` listed with its next scheduled run time.

---

### Step 4 — Test right now

Don't wait until tomorrow morning. Run it immediately:

```bash
openclaw cron run morning-briefing
```

Check your Telegram. The briefing should arrive within a few seconds.

## What you'll see

Without calendar and weather skills connected, your agent will work with what it knows — your USER.md context, the current date, and the standing instructions in SOUL.md. The response will be honest about its limitations but still useful.

A typical first briefing might read:

> **Thursday, March 27**
>
> Focus: Based on what you've shared, your main project is [X]. Today's a good day to push that forward.
>
> Reminder: No external calendar connected yet — add that in a future mission to get event summaries.

That's the automation working. The content improves as you add more context to USER.md and connect more tools.

## Confirm mission success

- `openclaw cron list` shows `morning-briefing` with a next run time
- Running `openclaw cron run morning-briefing` delivers a message to your Telegram

| Problem | Fix |
|---|---|
| `cron list` shows nothing | Config syntax error — validate your JSON at jsonlint.com |
| Command runs but Telegram gets nothing | Verify `"channel": "telegram"` matches the channel name in your config |
| Message arrives at wrong time | Set explicit `"timezone"` in the automation object |

## Bonus objective (+50 XP)

Add a second automation — an end-of-day summary. It should run at 6pm weekdays only and ask your agent to recap what you accomplished and suggest your top priority for tomorrow.

```json
{
  "id": "evening-recap",
  "schedule": "0 18 * * 1-5",
  "prompt": "Send me a brief end-of-day recap. What should I have gotten done today based on my context? What's the single most important thing for tomorrow? 3 lines max.",
  "channel": "telegram"
}
```

Add this to your `automations` array alongside `morning-briefing` and test with `openclaw cron run evening-recap`.
