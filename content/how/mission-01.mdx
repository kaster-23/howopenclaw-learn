## Tactical overview

By the end of this mission, your AI agent will be running on your own machine and responding to messages through a local web interface. Nothing leaves your device without passing through your config first. You own this.

You need:
- A terminal (macOS, Linux, WSL2, or Windows PowerShell)
- An API key from [Anthropic](https://console.anthropic.com), [OpenAI](https://platform.openai.com), or [Google AI Studio](https://aistudio.google.com)
- Node.js 22.14 or later (the installer checks this)

---

## Intel briefing

Most AI tools work the same way: you type something, it goes to a data center you don't control, gets processed, and comes back. You have no visibility into what happens in between.

OpenClaw works differently. The agent runs locally. The Gateway — the server that handles your messages — is a process on your machine. Your config, your personality files, your conversation context: all of it lives in `~/.openclaw/`. You can read every file. You can edit every setting. You can shut the whole thing down with one command.

That's what you're setting up in the next 15 minutes.

## Mission execution

### Step 1 — Install

**macOS / Linux / WSL2:**
```bash
curl -fsSL https://openclaw.ai/install.sh | bash
```

**Windows (PowerShell):**
```powershell
iwr -useb https://openclaw.ai/install.ps1 | iex
```

<Callout type="warn" title="Review before running">
Piping a remote script to bash executes code with your full user permissions. You can read the script at `https://openclaw.ai/install.sh` before running — always good practice.
</Callout>

The installer sets up Node.js if it isn't present and adds `openclaw` to your PATH. Verify it worked:

```bash
openclaw --version
```

If you see a version number, proceed. If you see `command not found`, restart your terminal and try again. If the installer failed on a Node version error:

```bash
# macOS/Linux — install Node 24 via nvm
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/HEAD/install.sh | bash
nvm install 24
```

Then re-run the installer.

---

### Step 2 — Run the setup wizard

```bash
openclaw onboard --install-daemon
```

The wizard asks four things:

1. **Your AI provider** — Choose one and enter your API key. If you don't have a key yet, create an account at your chosen provider and go to their API Keys section.
2. **Your workspace path** — Where files your agent creates will live. The default (`~/.openclaw/workspace`) is fine.
3. **Channel setup** — Skip for now. You'll connect Telegram in Mission 03.
4. **Daemon install** — Say yes. This starts the agent automatically when your machine starts.

When the wizard finishes, your config lives at `~/.openclaw/openclaw.json`.

---

### Step 3 — Verify the Gateway

If the daemon installed, the Gateway is already running. Check:

```bash
openclaw gateway status
```

You should see:

```
Gateway: running (pid 12847, uptime 0h 0m)
Channels:
  ✓ webchat      ready
Automations: 0 active
```

The key line is `Gateway: running`. If it's not running, start it manually:

```bash
openclaw gateway
```

---

### Step 4 — First contact

```bash
openclaw dashboard
```

This opens `http://localhost:18789` — your local web interface. Type a message and press Enter.

Your agent responds.

---

## Confirm mission success

Two things should be true:
1. `openclaw gateway status` shows `Gateway: running`
2. Your agent responded to your message in the dashboard

If both are true, Mission 01 is complete.

| Problem | Fix |
|---|---|
| `openclaw: command not found` | Restart terminal, or add npm's global bin to your `$PATH` |
| Node version error | Install Node 24 via `nvm install 24`, then re-run installer |
| Gateway won't start | Check logs: `openclaw logs` — usually a port conflict on 18789 |
| No response in dashboard | Verify API key is valid and has credit |

## Bonus objective (+50 XP)

Install via npm directly instead of the curl script:

```bash
npm install -g openclaw@latest
```

This works because OpenClaw is a Node.js package. The curl installer is a convenience wrapper around this command — it just handles Node version checks and PATH setup first. Understanding this means you know exactly what's on your machine.
