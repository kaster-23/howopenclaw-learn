<Callout type="warn" title="Do this before sharing access">
Your agent has shell access, can read your files, browse the web, and send messages on your behalf. Set DM policies before sharing any channel link or phone number.
</Callout>

<Steps>
<Step>

### Verify the daemon is running

If you ran `openclaw onboard --install-daemon`, your agent is already daemonized. Confirm:

```bash
openclaw gateway --status
```

You should see `running`. If not, start it:

```bash
openclaw gateway --port 18789
```

</Step>
<Step>

### Set your DM policy

OpenClaw has three DM policy tiers. Choose the right one for your use case:

| Policy | Who can reach your agent | When to use |
|--------|--------------------------|-------------|
| **Allowlist** | Only contacts you add | Personal agents — iMessage, WhatsApp |
| **Approval** | Anyone, but you approve first | Bots you want some oversight on |
| **Open** | Anyone, no restrictions | Dedicated public bots with other security layers |

Start with **allowlist** — you can always relax it later:

```bash
openclaw config set dm.policy allowlist
```

</Step>
<Step>

### Add a trusted contact

Add yourself so you can test messaging the agent:

```bash
openclaw contacts add --name "Me" --channel telegram --id YOUR_TELEGRAM_ID
```

Replace `telegram` with whichever channel you plan to use.

</Step>
<Step>

### Confirm the setup

Check your config is saved:

```bash
openclaw config get dm.policy
# → allowlist

openclaw contacts list
# → shows your contact
```

</Step>
</Steps>

<Callout type="info" title="Why DM policies matter">
Without access control, anyone who discovers your channel address could message your agent. The allowlist tier means only contacts you explicitly approve can interact with it — blocking prompt injection from unknown senders.
</Callout>
